# JamIssue 화면설계서

기준일: 2026-03-16  
기준 저장소: `D:/Code305/JamIssue`

이 문서는 PRD 방향과 실제 구현 사이의 화면 기준을 맞추기 위한 설계 문서다.
기능 추가나 화면 수정은 이 문서의 규칙을 먼저 따르고, 구현이 다르면 문서를 먼저 갱신한다.

## 1. 기본 원칙

- 모바일 우선
- 기준 폭은 `390px`
- 한 화면에서 전달하려는 핵심 행동은 1개를 우선한다.
- 상단 고정은 최소화하고, 하단 탭만 기본 고정으로 본다.
- 텍스트를 길게 읽게 하지 않는다.
- 지도와 카드 중심으로 빠르게 고르게 만든다.
- 기술 스택이나 개발자용 문구를 사용자 화면에 노출하지 않는다.

## 2. 전역 규칙

### 2-1. 레이아웃
- 좌우 패딩: `16px`
- 섹션 간 간격: `20px ~ 24px`
- 카드 간 간격: `12px ~ 16px`
- 하단 탭 높이: 터치 영역 포함 `64px` 이상
- 주요 버튼 높이: `44px` 이상

### 2-2. 고정 영역
- 기본 고정 영역은 하단 탭만 둔다.
- 상단 헤더는 본문 스크롤 안에 포함한다.
- 상태 배너, 브랜딩, 서브카피, 필터를 동시에 고정하지 않는다.

### 2-3. 타이포그래피
- 기본 폰트: `Pretendard`
- 제목: `20px`
- 본문/버튼: `12px`
- 캡션/배지: `8px`
- 제목은 기본 2줄 이내를 목표로 한다.
- 줄바꿈은 `keep-all` 성향을 우선한다.

### 2-4. 색과 분위기
- 메인 액션: 벚꽃 핑크 계열
- 보조 강조: 하늘색 계열
- 전체 톤: 밝은 파스텔
- 다크모드는 현재 범위에서 제외

### 2-5. 상태 표현
- 로딩: 가벼운 문구와 카드/맵 플레이스홀더 중심
- 비활성: 단순 회색 처리만 하지 말고 이유를 함께 보여준다.
- 성공/실패: 화면 전체를 덮기보다 해당 동작 근처의 짧은 메시지를 우선한다.

## 3. 전역 상호작용 규칙

- 1차 이동은 하단 탭 `탐색 / 코스 / 스탬프 / 마이`
- 2차 이동은 시트 또는 카드 확장
- 3차 이동이 필요하면 별도 화면으로 분리 검토
- 조건형 CTA는 기본 비활성 + 이유 노출

예시:
- 스탬프 버튼: 로그인 + 위치 확인 + 반경 충족 시 활성화
- 후기 작성 버튼: 로그인 + 정책 충족 시 활성화

| S03 | 후기/댓글 영역 | 방문 감상 공유와 상호작용 | 장소 상세 시트 내부 |

| S08 | 관리자 패널 | 장소 노출과 운영 요약을 다룬다 | 관리자 계정의 마이 탭 |

## 5. 화면별 기준

### S01 탐색 탭
목적:
- 지금 갈 만한 장소를 빠르게 고르게 한다.

필수 요소:
- 브랜드 헤더
- 짧은 서브카피
- 카테고리 필터 칩
- 대전 중심 지도
- 선택 장소 미리보기 카드
- 최신 후기 미리보기 0~2개

금지:
- 기술 스택 문구 노출
- 과한 상단 고정
- 상태 메시지가 화면 위를 오래 점유하는 구조

### S02 장소 상세 시트
목적:
- 장소 정보, 후기, 스탬프 상태를 한 번에 본다.

필수 요소:
- 장소명, 구, 예상 방문 시간
- 한 줄 요약과 설명
- 분위기 태그
- 스탬프 상태 카드
- 후기 리스트와 댓글 진입
- 후기 작성 CTA

규칙:
- 댓글/후기 영역은 같은 시트 안에서 이어지게 한다.
- 스탬프 상태와 후기 권한 문구를 섞어 모호하게 보이지 않게 한다.

### S03 후기/댓글
목적:
- 방문 감상을 카드 단위로 공유한다.

필수 요소:
- 후기 카드
- 감정 태그
- 작성 시각
- 이미지 0~1장
- 댓글 / 대댓글

정책:
- 댓글 직접 삭제는 soft delete
- 대댓글은 유지
- 후기 삭제 시 해당 피드 아래 댓글은 정리

### S04 코스 탭
목적:
- 운영자 코스와 사용자 생성 경로를 빠르게 비교한다.

필수 요소:
- 무드 필터 5종
- 코스 카드 리스트
- 사용자 경로 섹션
- 좋아요순 / 최신순 정렬

규칙:
- 필터 칩은 균일한 높이와 간격 유지
- 카드 헤더의 제목과 시간 배지는 같은 정렬 기준 유지

### S05 스탬프 탭
목적:
- 현장 방문 유도와 보상을 체감하게 한다.

필수 요소:
- 스탬프 진행 요약
- 현재 위치 확인 상태 카드
- 장소별 스탬프 카드
- 거리 안내
- 적립 CTA

규칙:
- 기본은 비활성 상태
- 위치 확인 전에는 버튼을 열지 않는다.
- 반경 안에 들어왔을 때만 활성화한다.
- 이전 장소 상태 문구가 다른 탭으로 흘러가지 않게 한다.

### S06 마이 탭
목적:
- 계정 상태와 내 기록을 본다.

필수 요소:
- 로그인 상태 블록
- 내 후기 수 / 내 스탬프 수 / 내 경로 수
- 내 후기 목록
- 내 스탬프 장소 목록
- 내 경로 작성기

규칙:
- 로그인 전과 후가 명확히 갈려야 한다.
- 로그인 성공 후에도 로그인 CTA가 남아 있으면 안 된다.

### S07 로그인 블록
목적:
- 계정 연결과 현재 세션 상태를 보여준다.

규칙:
- 현재 범위의 실구현 제공자는 네이버
- 카카오는 구조만 열려 있고 실제 OAuth는 후속 작업
- 로그인 성공 후에는 "지금 할 수 있는 행동"을 먼저 보여준다.

### S08 관리자 패널
목적:
- 장소 노출 여부와 운영 요약을 본다.

규칙:
- 일반 사용자에게 노출되면 안 된다.
- 향후 운영 기능이 늘면 별도 화면 분리 검토

## 6. 공통 컴포넌트 규칙

- 하단 탭은 항상 4개 고정
- 칩은 높이, 패딩, 폰트 크기를 계열별로 통일
- 카드 내부의 제목, 본문, 태그, CTA 순서를 통일
- 지도는 대전 범위 중심으로만 보여준다.

## 7. QA 체크리스트

- 제목이 비정상적으로 한 글자씩 꺾이지 않는가
- 관리자 UI가 일반 사용자에게 보이지 않는가
- 로그인 성공 후 로그인 CTA가 남아 있지 않은가
- 스탬프 CTA가 반경 밖에서도 활성화되지 않는가
- 거리/상태/오류 문구가 깨지지 않는가
- 코스 필터와 칩 간격이 통일되어 보이는가
- 상단 고정 영역이 과하지 않은가
- 390px 기준에서 카드 정렬이 무너지지 않는가

## 8. 다음 보강 우선순위

3. 관리자 화면 분리 여부 결정
4. 공공데이터 기반 실제 탐색 흐름 보강
5. 모바일 QA 결과를 이 문서에 누적
